### 二、 DRL 文件详细修改清单

#### 1. 头部声明修改

* **新增导入**：
```java
import com.example.ontology.model.UnemploymentPolicy;
import com.example.ontology.model.UnemploymentTimeline;

```


* **移除冗余**：如果之前为了处理日期导入了 `java.util.Date` 等，现在可以移除，逻辑应下沉到 `UnemploymentTimeline`。

#### 2. 规则逻辑模式变换 (Pattern Change)

| 规则维度 | 修改前 (RDBMS/硬编码模式) | **修改后 (本体论/参数化模式)** |
| --- | --- | --- |
| **参数引用** | `paidMonths >= 12` | `$policy: UnemploymentPolicy()`, <br>

<br>`InsuranceRecord(paidMonths >= $policy.minContributionMonths)` |
| **临近退休** | `(60 - age) <= 5` | `$p: UnemployedPerson(isNearRetirement($policy) == true)` |
| **离职原因** | `stopReason == "非本人意愿"` | `InsuranceRecord(isInvoluntaryStop == true)` |
| **最后参保地** | 在 DRL 中用日期比对找最后一条 | `InsuranceRecord(this == $timeline.latestRecord, isLastInsuredInShenzhen == true)` |

#### 3. 典型规则补全模板（以情形三为例）

修改后的 DRL 结构应如下所示：

```drl
rule "Unemployment_Scenario_3_NearRetirement"
    when
        // 1. 获取当前生效的政策单例
        $policy : UnemploymentPolicy()
        
        // 2. 判定个人是否符合临近退休条件（逻辑已封装在 Person 类中）
        $p : UnemployedPerson(
            isDomicileShenzhen == true, 
            isNearRetirement($policy) == true  // 调用动态参数进行判定
        )
        
        // 3. 通过时间线对象锁定最后一段参保事实
        $timeline : UnemploymentTimeline(personId == $p.id)
        $latest : InsuranceRecord(
            this == $timeline.latestRecord, 
            isInvoluntaryStop == true,         // 已脱敏的布尔判定
            isLastInsuredInShenzhen == true    // 预计算的地理判定
        )
        
        // 4. 资格前提：满足月数要求或有剩余期限
        eval($timeline.getTotalMonths() >= $policy.getMinContributionMonths() || $p.hasRemainingEntitlement())
    then
        insert(new EligibilityResult($p.getId(), "ELIGIBLE_SCENARIO_3", "符合临近退休人员申领条件"));
end

```

### 三、 检查与验收准则 (Checklist)

在 Copilot 修改完 DRL 后，请核对以下几点：

1. **全局搜索 `12`、`60`、`55**`：如果 DRL 里还能搜到这些数字，说明**动态参数化**补全失败。
2. **全局搜索 `"中文字符"**`：如果 DRL 里还有对 `stopReason` 或 `unitType` 的中文比对，说明**语义化脱敏**补全失败。
3. **检查 `eval` 函数**：尽量减少 `eval` 中的复杂运算。如果发现 DRL 在做 `Date` 减法，说明**时间维度控制**没有正确下沉到本体层。
4. **单例匹配**：确保规则第一行匹配了 `$policy : UnemploymentPolicy()`，否则规则将因为缺少事实而永远无法触发。

通过这些修改，你的 `.drl` 文件将变得极其简洁，且能够自适应未来《操作规程》中退休年龄或缴费门槛的任何变动。
